## Focus Length（像素）计算方法
结合你这款摄像头的参数，两个参数直接按下面填即可，附完整推导过程和微调方法。

### 一、参数直接填写结果

python

运行

```
BASELINE = 65.0        # 基线，直接和参数对应，不用换算
FOCAL_LENGTH = 800.0   # 像素焦距，基于1/2.7"传感器+3.6mm物理焦距+1280px单目宽度推算
```

---

### 二、详细推导说明

#### 1. BASELINE（基线）

参数页明确标注「基线 65mm（左镜头中线到右镜头中线距离）」，和代码里的`BASELINE`含义完全一致，单位都是毫米，**直接填入 65.0 即可**，无需任何换算。

这个值决定深度计算的比例基准，基线越大，远距离深度精度越高。

#### 2. FOCAL_LENGTH（像素焦距）

⚠️ 注意：**不能直接填 3.6**。3.6mm 是镜头的**物理焦距（毫米单位）**，而代码里需要的是**像素焦距（像素单位）**，两者需要通过传感器尺寸和输出分辨率换算。

##### 换算依据

- 传感器规格：1/2.7 英寸，16:9 比例，行业标准像素尺寸为 3μm（3 微米）
- 传感器物理宽度：全像素 1920px × 3μm = 5.76mm
- 你的设置：总分辨率 2560×720 → 单目宽度 = 2560 ÷ 2 = 1280px
- 物理焦距：3.6mm

##### 计算公式

$\text{像素焦距} = \frac{\text{物理焦距(mm)} \times \text{单目图像宽度(px)}}{\text{传感器物理宽度(mm)}}$

代入数值计算：

$\text{FOCAL\_LENGTH} = \frac{3.6 \times 1280}{5.76} = 800 \ \text{像素}$

---

### 三、精度验证与微调方法

这个 800 是理论推算值，误差在 5% 以内，**完全满足「深度方差校验修正风扇 / 钟表误检」的需求**（方差看相对波动，绝对值误差不影响判断）。

如果需要更精准的测距，可以按以下方法快速校准：

1. 把摄像头对准一个已知距离的平面物体（比如距离镜头 500mm 的书本 / 墙面）
2. 运行代码查看检测框输出的深度数值
3. 按比例微调：
    
    - 测出的深度比真实值大 → 调大`FOCAL_LENGTH`
    - 测出的深度比真实值小 → 调小`FOCAL_LENGTH`
        
        例如真实 500mm，代码测出 550mm，就把 800 改成 800 × (550/500) = 880

### 四、补充说明

1. **视场角交叉验证**
    
    参数页标注水平视场角 80°，用视场角公式反推像素焦距约为 763 像素，和 800 的推算值差异约 5%，属于镜头标称公差的正常范围，不影响误检修正功能。
    
2. **分辨率变化对应调整**
    
    如果你后续改了总分辨率，像素焦距要等比例缩放。比如改成总 1280×480（单目 640px 宽），`FOCAL_LENGTH`就要改成 800 × (640/1280) = 400。
    
3. **最精准方案**
    
    如果后续需要工业级测距精度，用棋盘格做一次双目立体标定，标定结果内参矩阵里的`fx`值就是最精准的像素焦距，直接替换即可。
